## Description
Integrate the `karafka` framework to enable asynchronous message processing. This task involves setting up the producer side by refactoring the event ingestion endpoint to publish events to Kafka instead of saving them directly to the database.

---

## Acceptance Criteria
- [ ] `karafka` gem is successfully installed and initialized.
- [ ] `POST /api/v1/events` publishes a message to the `user.events` topic.
- [ ] The API response for successful ingestion is `{ "status": "queued" }` with a `202 Accepted` or `201 Created` status code (as per requirement, though "queued" often implies 202).
- [ ] Karafka configuration correctly points to the Kafka brokers defined in environment variables.

---

## Implementation Checklist
- [ ] Add `karafka` to `Gemfile` and run `bundle install`.
- [ ] Run `bundle exec karafka install` to generate configuration files.
- [ ] Configure `karafka.rb` to use `KAFKA_BROKERS` environment variable.
- [ ] Define the `user.events` topic in `karafka.rb`.
- [ ] Refactor `Api::V1::EventsController#create` to use `Karafka.producer.produce_async`.
- [ ] Update controller response to return `status: "queued"`.
- [ ] Add/update tests to verify that the message is produced to Kafka (using Karafka testing tools or mocks).

## Step-by-step Guide

**Files to create/modify:**
- `Gemfile` — Add `karafka`.
- `karafka.rb` — Primary configuration and routing.
- `app/controllers/api/v1/events_controller.rb` — Refactor to publish events.
- `test/integration/api/v1/event_ingestion_test.rb` — Update expectations.

**Key decisions:**
- Use `produce_async` for high-throughput ingestion without blocking the web request.
- Use `202 Accepted` for the response status to reflect that the processing is now asynchronous.

**Non-obvious snippets**:
```ruby
# app/controllers/api/v1/events_controller.rb
def create
  # ... validation ...
  Karafka.producer.produce_async(
    topic: 'user.events',
    payload: event_params.merge(user_id: current_user.id).to_json
  )
  render json: { status: 'queued' }, status: :accepted
end
```
