## Description
Implement the registration and login endpoints to allow users to create accounts and obtain JWTs for authenticated requests.

---

## Acceptance Criteria
- [ ] `POST /auth/register` creates a user with valid email and password.
- [ ] `POST /auth/register` returns 201 Created on success and 422 Unprocessable Entity on failure.
- [ ] `POST /auth/login` returns a valid JWT on success and 401 Unauthorized on failure.
- [ ] JWT contains the `user_id` and expires in 24 hours.
- [ ] Integration tests cover successful and failed scenarios for both endpoints.

---

## Implementation Checklist
- [ ] Create `AuthController` (or similar) to handle auth requests.
- [ ] Implement `register` action: create user, return success/error response.
- [ ] Implement `login` action: authenticate user, generate JWT using `JwtService`, return response.
- [ ] Define routes in `config/routes.rb` under an `/auth` namespace or direct routes.
- [ ] Add integration tests in `test/integration/authentication_test.rb`.

---

## Step-by-step Guide

**Files to create/modify:**
- `app/controllers/auth_controller.rb` — Create the controller for registration and login.
- `config/routes.rb` — Add routes for `POST /auth/register` and `POST /auth/login`.
- `test/integration/authentication_test.rb` — Integration tests for the auth flows.

**Key decisions:**
- Use `User#authenticate` provided by `has_secure_password`.
- Use `JwtService.encode` to generate tokens.
- Keep the response JSON consistent (e.g., `{ token: '...' }` or `{ error: '...' }`).

**Non-obvious snippets**:
```ruby
# config/routes.rb
post 'auth/register', to: 'auth#register'
post 'auth/login', to: 'auth#login'
```
