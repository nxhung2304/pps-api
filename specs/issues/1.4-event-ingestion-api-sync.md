## Description
Implement a synchronous event ingestion endpoint that allows authenticated users to post event data. The endpoint will validate the incoming payload and save it directly to the PostgreSQL database. This serves as the "Walking Skeleton" for event data flow before introducing Kafka.

---

## Acceptance Criteria
- [ ] `POST /api/v1/events` requires a valid JWT.
- [ ] Returns `201 Created` on successful save.
- [ ] Validates event `type` and `timestamp`.
- [ ] Saves event `payload` as JSONB.
- [ ] Returns `401 Unauthorized` for missing or invalid tokens.
- [ ] Returns `422 Unprocessable Entity` for invalid payloads.
