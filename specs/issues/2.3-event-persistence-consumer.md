## Description
Implement the `EventPersistenceConsumer` to consume messages from the `user.events` Kafka topic and save them to the PostgreSQL `events` table. This enables asynchronous event persistence.

---

## Acceptance Criteria
- [ ] `EventPersistenceConsumer` successfully consumes messages from the `user.events` topic.
- [ ] Consumed messages are correctly deserialized and saved to the `events` table.
- [ ] End-to-end async persistence is verified (POST to API -> Kafka -> Consumer -> DB).
- [ ] Proper error handling for malformed messages or DB failures.

## Step-by-step Guide

**Files to create/modify:**

**Key decisions:**

- Log failures to help with debugging but don't stop the consumer for a single bad message.

**Non-obvious snippets**:
