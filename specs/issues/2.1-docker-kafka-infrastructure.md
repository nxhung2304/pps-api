## Description
Set up the core messaging infrastructure using Docker Compose. This provides the backbone for asynchronous event processing, allowing the API to decouple ingestion from persistence and processing.

## Step-by-step Guide

**Files to create/modify:**
- `docker-compose.yml` — Add Zookeeper and Kafka services.
- `README.md` — Document how to start and verify Kafka.

**Key decisions:**
- Use Confluent Community images (`cp-zookeeper`, `cp-kafka`) as they are industry standard.
- Use a single-node Kafka setup for development.
- Expose Kafka on port `9092` for internal Docker traffic and `29092` for host traffic (optional but helpful).

**Non-obvious snippets**:
```yaml
  kafka:
    image: confluentinc/cp-kafka:7.5.0
    depends_on:
      - zookeeper
    volumes:
      - ./tmp/kafka/data:/var/lib/kafka/data
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:9092,PLAINTEXT_HOST://localhost:29092
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_INTER_BROKER_LISTENER_NAME: PLAINTEXT
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
```

---

## Notes
- Ensure you have enough memory allocated to Docker (Kafka can be heavy).
- Zookeeper is required by older Kafka versions/images; while newer Kafka (KRaft) doesn't need it, `cp-kafka` still defaults to Zookeeper in many stable versions.
